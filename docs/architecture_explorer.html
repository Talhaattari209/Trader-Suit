Non-Functional Requirements
Performance
  • All compute caps enforced via ComputationBudget — local runs finish in seconds
  • MC simulation is vectorised NumPy (no Python loops per path)
  • RL training capped at CB.rl_timesteps to avoid OOM on T4
  • Dashboard calls are lightweight HTTP — heavy work is async in agents
Reliability
  • All optional imports wrapped in try/except ImportError — SB3, torch, alpaca all degrade gracefully
  • MC route falls back to _synthetic_mc_results() if MonteCarloPro throws
  • FilesystemStore uses atomic JSON writes; agents re-scan on next poll if file corrupt
  • BaseWatcher poll loop catches all exceptions and continues
Scalability
  • Multiple strategy drafts handled independently (filename-based isolation)
  • edge_registry.py allows new edge modules without touching agents
  • Additional LLM providers added by subclassing BaseLLMClient
  • Additional watchers added by subclassing BaseWatcher
Security
  • API keys in .env — never committed (in .gitignore)
  • Live trades require manual file move to Approved/ (HITL gate)
  • CORS configured on FastAPI — restrict origins in production
  • Zero-MQL: all execution logic in Python only
